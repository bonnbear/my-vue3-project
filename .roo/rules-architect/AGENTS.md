# Project Architecture Rules (Non-Obvious Only)

- **單體狀態機架構**: 核心架構是一個在 `src/App.vue` 中實現的單體狀態機。任何架構決策都必須考慮到所有邏輯都高度耦合在此單一檔案中。
- **避免拆分檔案**: 專案的既定模式是將子元件保留在 `src/App.vue` 內部。除非有極其充分的理由，否則不要將元件拆分到新的 `.vue` 檔案中，這會破壞現有架構。
- **渲染函數優先**: 新的 UI 元件應優先考慮使用渲染函數 (`h()`) 建立，以保持與現有內部元件定義的一致性。
- **無中心化狀態**: 由於所有狀態都是本地的，因此不存在跨元件共享狀態的架構問題。但這也意味著所有狀態操作都必須在 `src/App.vue` 中進行。
